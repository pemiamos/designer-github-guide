## 收藏管理建议

GitHub 自带了 Star List 功能,登录后进入 **Your stars** 页面可以新建多个列表。建议按以下方式分类:

- `字体` / `图标` / `设计系统` / `可视化` / `设计工具` / `AI 工具`

也可以考虑建一个公开仓库叫 `design-resources`,在 README 里维护自己的私人资源索引。这种做法的好处是可以写上你为什么推荐某个项目,比单纯的 Star 更有传承价值。
