# 三种使用方式

对设计师而言,进入 GitHub 有三条由浅入深的路径。不必一开始就学命令行——大部分设计师的日常需求,用前两种方式就能覆盖。

## 方式一:纯浏览器(零门槛)

这是最容易被低估的一种方式。仅靠浏览器和 GitHub 网页,你就能:

- 浏览任何公开项目的全部文件
- 下载整个项目(Code 按钮 → Download ZIP)
- 下载发布版本(Releases 页面)
- Star 收藏,Fork 复制,Issue 提问
- 甚至直接在网页上编辑文件并提交

### 搜索技巧

- 搜索设计资源常用关键词:`icon set free`、`design system open source`、`variable font`、`figma plugin`
- 按语言筛选:点 `Languages` 过滤,选 `HTML` / `CSS` / `TypeScript` 之类
- 按活跃度筛选:`Sort by recently updated` 能避开年久失修的项目

### Trending 页面

访问 `github.com/trending` 可以看到当下最热门的项目,按日/周/月切换。设计系统、UI 框架、图标库经常登榜。这是最简单的行业动态追踪方式。

## 方式二:GitHub Desktop(图形界面)

一款免费的官方图形客户端,下载地址 `desktop.github.com`,Mac 和 Windows 都有。

### 核心功能

- 一键 Clone 任何仓库到本地
- 一键同步本地改动到云端
- 可视化看到每一次改动的 diff 对比
- 切换分支像切换文件夹一样简单
- 完全不需要记任何命令

### 适合的场景

- 维护自己的设计文档或知识库
- 克隆开源项目到本地查看(比下载 ZIP 方便,因为能持续更新)
- 参与协作项目,但又不想深入学习命令行
- 对大多数独立设计师来说,这是**日常主力工具**

## 方式三:命令行(进阶)

学习曲线最陡,但灵活性最高。前两种方式解决不了的场景才需要它。

常用命令速查:

```bash
# 克隆远程仓库到本地
git clone https://github.com/用户名/项目名

# 进入项目目录
cd 项目名

# 查看当前状态(哪些文件改了、哪些未提交)
git status

# 拉取远程最新更新
git pull

# 提交本地改动
git add .
git commit -m "说明这次改动"
git push
```

### 什么时候值得学命令行

说实话,对大多数设计师来说,**暂时可以不学**。但以下几种情况会让命令行变得值得:

- 你开始维护超过 3 个仓库,图形界面切换效率降低
- 你要用到 Claude Code、Cursor 等 AI 编码工具,它们都基于命令行
- 你要做自动化,比如定时备份、批处理任务
- 你要和程序员同事合作,对方希望你用标准工作流

## 选择建议

不要一开始就追求"标准的工程师工作流"。从方式一开始,逐步过渡到方式二,等到确实遇到方式二解决不了的问题,再考虑方式三。

**工具要服务于工作,而不是反过来。**
