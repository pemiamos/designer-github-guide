# 设计师的 GitHub 指南

> 一份由设计师写给设计师的 GitHub 入门与进阶指南。  
> 不谈代码,只谈设计师如何把 GitHub 变成自己的工作基础设施。

## 这份指南是什么

作为一名工业设计与交互设计的教师,我在把 AI 工具、版本管理、知识库系统整合进日常创作与教学的过程中,发现设计师进入 GitHub 存在一个普遍的认知鸿沟——现有教程几乎全部为工程师而写,语言、案例、思维方式都带着工程师的默认前提。

这份指南从设计师的真实需求出发,回答一个核心问题:**设计师如何让 GitHub 成为自己日常工作的一部分?**

## 为什么要做这件事

设计师群体在 GitHub 上的存在感远低于工程师,不是因为设计师不够技术,而是因为:

1. **入门门槛被高估**——实际上浏览器加 GitHub Desktop 就能覆盖 90% 的使用场景
2. **思维模型被照搬**——工程师的版本管理逻辑不完全适合设计师的工作流
3. **价值叙事被误导**——以为 GitHub 只能放代码,忽视了它作为"设计文档基础设施"的潜力

这份指南试图提供另一种叙事。
